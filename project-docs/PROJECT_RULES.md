# PROJECT RULES

## 핵심 방향

- simple is best
- 최대한 단순한 UX 유지
- 클릭 수 최소화
- 입력 피로도 최소화
- 복잡한 기능보다 직관 우선

---

## UI 방향

- 깔끔한 SaaS 스타일 유지
- 과도한 색상 사용 금지
- 카드/버튼 과대 디자인 금지
- 폰트 크기 과도하게 키우지 않기
- ERP 느낌 금지
- 현재 ZETA 톤 유지

---

## 개발 원칙

- 부분 패치 금지
- 기능 수정 시 항상 전체 스크립트 기준 제공
- 기존 기능 훼손 금지
- 기능 추가 전 UX 우선 검토
- 모바일/PC 분리 고려

---

## 현재 개발 전략

### 기존 기능

현재 구현 완료된 기능은 안정성 우선.

- 업무일지
- 일정관리
- 로그인
- 기존 CRUD 기능

당분간 전면 리팩토링 금지.

필요 시:
- Header
- Button
- Modal

등의 소규모 UI 컴포넌트만 부분 분리 가능.

---

### 신규 기능

앞으로 추가되는 기능은 처음부터 모듈화 기준으로 개발.

예:
- components
- hooks
- types

구조 기반으로 작성.

page.tsx 비대화 금지.

---

## 일정관리 방향

- 달력은 최대한 심플하게 유지
- 이름만 표시
- 상세 내용은 클릭 시 확인
- 일정 종류 기반 색상 사용
- 출장은 기간 등록 지원
- 본인 일정만 수정/삭제 가능

---

## 업무일지 방향

- 전일 / 금일 분리 유지
- 문서형 조회 유지
- 가독성 최우선
- 한 화면에서 흐름 확인 가능해야 함
- 수정/삭제 가능 구조 유지

---

## 향후 방향

- 결재 시스템 연동 예정
- 휴가계획서 → 일정관리 자동 반영 예정
- 업무일지 외근 데이터 연동 가능성 있음
- 권한 시스템 세분화 예정
