# REFACTOR RULES

## 현재 방향

- 기존 기능 최대한 유지
- 전면 리팩토링 금지
- 신규 기능부터 모듈화
- 기존 기능은 필요 시 점진 분리

---

## 현재 레거시 영역

- main/page.tsx
- schedule/page.tsx
- view/page.tsx

현재는 안정성 우선.

---

## 신규 기능 개발 규칙

- page.tsx 비대화 금지
- components 분리 우선
- hooks 기반 로직 분리
- 공통 UI 재사용

---

## 현재 허용 리팩토링 범위

### 가능

- Header 분리
- 공통 Button 분리
- 공통 Modal 분리
- 단순 UI 컴포넌트 분리

---

### 당분간 금지

- 일정 CRUD 전체 수정
- 로그인 흐름 수정
- 권한 구조 변경
- 복잡한 useEffect 수정
- 전체 상태 구조 변경

---

## 장기 방향

- 신규 구조를 기준 구조로 성장
- 기존 레거시는 점진 교체
- 기능 안정성 우선 유지
